环境搭建

根据自身情况，此处选择WSL+VS Code作为基本开发环境

WSL安装教程

WSL：The Windows Subsystem for Linux lets developers run a GNU/Linux environment – including most command-line tools, utilities, and applications – directly on Windows, unmodified, without the overhead of a traditional virtual machine or dual-boot setup.

Note：WSL分为WSL1和WSL2，WSL2底层使用虚拟化技术，要求系统版本在1903及以上，此处选择WSL1

  1. 管理员运行power shell，输入dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart，重启计算机
  2. 下载Debian 9
  3. 执行add-appxpackage filename进行安装，之后按提示设置用户名和密码
  4. 进行换源操作，使用清华大学开源镜像站

VS Code安装教程

  1. 下载VS Code并安装
  2. 安装WSL扩展，在工作目录执行code .即可

node.js安装教程

  1. node.js提供了多种安装方式，访问官网获取详情，此处使用包管理器进行安装
  2. Linux下安装git,curl等软件包，再执行以下命令：
    # Using Debian, as root
    1. curl -fsSL https://deb.nodesource.com/setup_12.x | bash -
    2. apt install nodejs
  3. 安装过程中已附带安装npm，对npm进行换源操作，npm config set registry https://registry.npm.taobao.org，再输入npm config list进行验证

hexo配置教程

  1. npm install -g hexo-cli
  2. 新建博客文件夹
  3. 进入该文件夹，运行hexo init进行初始化
  4. 在站点配置文件中编辑deploy，之后再运行npm install hexo-deployer-git --save
  5. hexo s/server运行本地服务器；hexo g/generate生产静态页面；hexo d/deploy部署到Github Pages
